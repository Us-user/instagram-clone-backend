## Решения
- **Разделение слоёв для SignalR: абстракция `IChatNotifier` в Infrastructure, реализация в WebApi.**
  Сервисы живут в Infrastructure, а `ChatHub`/`IHubContext` — в WebApi (по CLAUDE.md); Infrastructure
  не может ссылаться на WebApi. Поэтому `ChatService` зависит от `IChatNotifier` (Domain-DTO на входе),
  а `ChatNotifier` поверх `IHubContext<ChatHub, IChatClient>` регистрируется в WebApi. Чисто, без
  протечки SignalR-типов в бизнес-слой.
- **Адресация по userId через кастомный `IUserIdProvider`.** Токен кладёт id в claim `userId`
  (`CustomClaims.UserId`), а `DefaultUserIdProvider` читает `ClaimTypes.NameIdentifier`. Свой
  провайдер возвращает `userId`, что позволяет слать сообщения через `Clients.Users(...)` без
  ручного управления группами.
- **Рассылка обоим участникам, не только получателю.** `send-message` уведомляет `user1` и `user2`:
  получатель видит входящее, другие устройства отправителя синхронизируются.
- **Дедуп чата через нормализацию пары.** Уникальный индекс — на упорядоченной паре
  `(User1Id, User2Id)`; сервис сортирует участников `string.CompareOrdinal` перед поиском/вставкой,
  поэтому пара всегда отображается в один кортеж (совпадает с комментарием в `ChatConfiguration`).
- **`get-chat-by-id` помечает прочитанным только входящие.** `SenderUserId != current && !IsRead`
  — свои сообщения не трогаем; сохранение отдельным `SaveChanges` до проекции, чтобы вернуть уже
  актуальные `IsRead`.
- **Владелец/участник — `ForbiddenException` (403), отсутствие — `NotFoundException` (404).**
  Единообразно с Phase 6–7 (правило ТЗ №4). Query/route-параметры `int?`/`string?` + guard'ы —
  как в прошлых фазах (nullable + `BadRequestException` через exception-middleware в `Response<T>`).
- **Вложение сообщения сохраняется дефолтами `IFileService` (изображения, ≤10 МБ).** Контракт не
  уточняет тип «File»; для IG-клона вложения — картинки, отдельный набор расширений не вводил.
- **`LastMessage` берёт текст последнего сообщения** (для файлового сообщения текст = null —
  клиент отрисует «вложение»). Достаточно для превью в списке чатов.
