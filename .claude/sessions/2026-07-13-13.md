## Сделано
- **Поддержка `DATABASE_URL` (для Render/PaaS)** в `Infrastructure/DependencyInjection.cs`:
  метод `ResolveConnectionString` — если задан env/config `DATABASE_URL` в URL-формате
  (`postgres://user:pass@host:port/db`), конвертирует его в key-value Npgsql через
  `NpgsqlConnectionStringBuilder` и включает `SSL Mode=Require` (Render требует SSL; в Npgsql 8
  режим Require шифрует без строгой проверки сертификата, `Trust Server Certificate` не нужен).
  Иначе — прежняя `ConnectionStrings:DefaultConnection`. Секреты в git не попадают (URL
  передавали инлайн-переменной окружения в `dotnet run`).
- **Найден и исправлен реальный баг (`Domain/Entities/User.cs`):** `User : IdentityUser<string>`.
  Generic `IdentityUser<TKey>` (в отличие от не-generic `IdentityUser`) **не генерирует** строковый
  `Id` в конструкторе, а EF строковый первичный ключ не автогенерит → при создании пользователя
  `Id == null` → `InvalidOperationException: Unable to track an entity of type 'User' because its
  primary key property 'Id' is null`. Падали **и seed, и обычная регистрация**. Раньше не всплывало —
  живой БД не было (все прошлые smoke — только swagger без БД). **Фикс:** добавлен конструктор
  `User()`, задающий `Id = Guid.NewGuid().ToString()` и `SecurityStamp` (как делает не-generic
  `IdentityUser`); при чтении из БД EF перезаписывает эти значения.

## Решения
- **Смоук гоняли до деплоя — и не зря.** Первый же запуск на живой БД поймал баг с `Id`, который
  бы гарантированно сломал регистрацию и seed в облаке. Это подтверждает выбранный порядок
  (сначала проверка локально на реальной БД, деплой — потом).
- **Баг чиним в сущности, а не в местах создания.** Конструктор `User()` — единая точка: покрывает
  и seed (`DbInitializer`), и регистрацию (`AccountService`), и любые будущие создания. Согласуется
  с тем, как устроен не-generic `IdentityUser`.
- **400 на add-comment с кириллицей — артефакт клиента, не бага.** PowerShell 5.1
  `Invoke-RestMethod` со строковым `-Body` без `charset=utf-8` шлёт не-UTF-8 байты → битый JSON →
  400. С `charset=utf-8`/ASCII — 200. Приложение UTF-8 обрабатывает корректно (seed хранит
  кириллицу «Закат над океаном 🌅»). В скрипте добавлен `charset=utf-8`.
- **Деплой отложен** по решению пользователя: впереди доп. фичи сверх ТЗ, web service поднимем
  в самом конце (`Dockerfile` + `render.yaml`).
