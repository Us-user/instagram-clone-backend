## Решения
- **Сторис по id → 404 (а не 403) при недоступности.** Единообразно с сокрытием факта
  существования приватного контента; по юзеру — пустая лента, как в остальных списковых выдачах.
- **`GetStoriesAsync`: фикс через `Status == Accepted`, без добавления `VisibleTo`.** Лента сторис
  собирается из подписок и не имеет `VisibleTo`-бэкстопа (в отличие от `get-following-post`);
  корректный фильтр здесь — именно статус подписки (pending к приватному не даёт доступ). Блок уже
  исключён, т.к. блокировка снимает подписку в обе стороны.
- **Сид не расширял.** Роли, приватный аккаунт, группа, уведомления/хэштеги/2FA уже наполнены в
  фазах 12–20; §13 требует «пример приватного аккаунта/группы» — они есть. Новых пробелов нет.
- **Swagger-теги не трогал.** Конвенция «тег = контроллер» + class-level XML `<summary>` уже
  удовлетворяет §13.9 (описания + Bearer), явные `[Tags]`/`SwaggerOperation` избыточны.
