# Логи сессий

Один файл на одну рабочую сессию. Именование: `YYYY-MM-DD-NN.md` (NN = 01, 02… в пределах дня).

- `/start` читает последние логи, чтобы возобновить работу, и пишет новый при завершении.
- `/stop` пишет/дополняет лог текущей сессии и коммитит.

Сортируй по имени файла — самый свежий лог последний по алфавиту.

## Шаблон

```markdown
---
date: YYYY-MM-DD
session: NN
phase: <фаза роадмапа>
---

## Цель сессии
<что планировалось сделать>

## Сделано
- ...

## Решения
- <важные технические решения и почему>

## Затронутые файлы
- ...

## Состояние сборки
<зелёная / сломана — детали>

## Открытые TODO
- ...

## Следующие шаги (для /start)
- ...
```
