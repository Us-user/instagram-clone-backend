## Цель сессии
Полностью закрыть Phase 1: все доменные типы компилируются — enum, обёртки ответов,
17 сущностей БД, контрактные DTO (дословно) и read/response-DTO под группы эндпоинтов.

## Сделано
- **Enum:** `Domain/Enums/Gender.cs` (`Male=0, Female=1` — жёстко по контракту).
- **Обёртки ответов:** `Domain/Responses/Response.cs` (`Data`/`Errors`/`StatusCode` + конструкторы
  успех/ошибка) и `PagedResponse.cs` (`PageNumber`/`PageSize`/`TotalRecords`/`TotalPages`).
- **17 сущностей** в `Domain/Entities/`:
  - `User : IdentityUser<string>` (+ `FullName`, `Avatar?`, навигации), `UserProfile` (1:1, `Gender`, `Image?`).
  - Посты: `Post` (+ `IsReel`, навигации Images/Likes/Comments/Views/Favorites),
    `PostImage`, `PostLike`, `PostView`, `PostComment`, `PostFavorite`.
  - Сторис: `Story` (`FileName?`, `PostId?`, живёт 24ч), `StoryLike`, `StoryView` (`ViewUserId`).
  - Прочие: `FollowingRelationShip` (`UserId` подписчик / `FollowingUserId` цель),
    `Chat` (`User1Id`/`User2Id`), `Message` (`MessageText?`/`FileName?`/`IsRead`),
    `Location`, `SearchHistory`, `UserSearchHistory`.
- **Контрактные DTO (дословно):** `RegisterDto`, `LoginDto`, `AddLocationDto`, `UpdateLocationDto`,
  `AddPostCommentDto`, `AddPostFavoriteDto`, `UpdateUserProfileDto`, `GetStoryDto`, `ViewerDto`,
  `GetStoryViewDto` — включая опечатку контракта `CreateAt` в `GetStoryDto`.
- **Read/response DTO по группам:** Post (`AddPostDto` multipart, `GetPostDto` со счётчиками+isLiked/isFavorite,
  `GetPostCommentDto`), UserProfile (`GetUserProfileDto` со счётчиками+isFollowing),
  User (`GetUserDto`, `GetSearchHistoryDto`, `GetUserSearchHistoryDto`),
  Location (`GetLocationDto`), Story (`GetStoriesDto`+`StoryItemDto` для сгруппированной ленты),
  Chat (`SendMessageDto` multipart, `GetMessageDto`, `GetChatDto`, `GetChatByIdDto`).

## Решения
- **Имена контракта сохраняем дословно**, даже опечатки: `GetStoryDto.CreateAt` (не `CreatedAt`).
  Опечатка `massageId` в delete-message — это query-параметр, а не DTO, поэтому появится в контроллере (Phase 8).
- **Свойство `ViewerDto ViewerDto` внутри `GetStoryDto`** — имя свойства совпадает с именем типа; в C# допустимо, оставил как в контракте.
- **`IFormFile`/`List<IFormFile>` в DTO** (`AddPostDto`, `SendMessageDto`) берём из `FrameworkReference`
  `Microsoft.AspNetCore.App`, уже подключённого в Domain (Phase 0) — отдельный пакет не нужен.
- **Навигации только для «безопасных» связей.** У `User` оставил коллекции постов/лайков/… и `SearchHistories`.
  Двусторонние FK на `User` (Chat.User1/User2, Following.User/FollowingUser, StoryView.ViewUser,
  UserSearchHistory) объявлены как одиночные навигации без обратных коллекций — обратные связи и
  каскады донастрою явно во Fluent API (Phase 2), чтобы избежать множественных cascade-путей.
- **Read-DTO для агрегатов (лента сторис, список чатов)** сделал по разумной форме сейчас;
  точную структуру уточню при реализации фич (Phase 6–8). Отмечено в ROADMAP.
