## Цель сессии
Полностью закрыть Phase 3: JWT-токен-сервис и подключение JWT-аутентификации/авторизации,
глобальный exception-middleware → `Response<T>`, file storage service, базовые профили
AutoMapper, регистрация FluentValidation с базовыми валидаторами, Swagger с Bearer-кнопкой
и XML-комментариями, доступ к текущему юзеру из claims.

## Сделано
- **Доменные исключения** `Domain/Exceptions/`: `BadRequestException` (400), `NotFoundException`
  (404), `ForbiddenException` (403) — общий словарь ошибок для сервисов и middleware.
- **JWT.** `Infrastructure/Options/JwtOptions.cs` (биндинг секции `Jwt`), `Infrastructure/Constants/
  CustomClaims.cs` (`userId`/`userName`/`email`; роль — стандартным `ClaimTypes.Role`).
  `ITokenService`/`TokenService` — HMAC-SHA256 JWT с claim'ами userId/userName/email/роли + `jti`.
- **Текущий пользователь.** `ICurrentUserService`/`CurrentUserService` читает claim'ы из
  `IHttpContextAccessor.HttpContext.User` (`UserId`, `UserName`, `Email`, `IsInRole`,
  `GetRequiredUserId()` → 401 если аноним). ID берётся из claims, не из параметров.
- **Файлы.** `IFileService`/`FileService` — сохранение в `wwwroot/images` под Guid-именем,
  проверка расширения (по умолчанию изображения) и размера (по умолчанию 10 МБ, параметры
  переопределяемы), удаление файла с диска; при нарушениях бросает `BadRequestException`.
- **AutoMapper.** `Infrastructure/Mapping/MappingProfile.cs` — базовые маппинги Location↔DTO,
  User→GetUserDto, UserProfile→GetUserProfileDto, Post→GetPostDto, PostComment→GetPostCommentDto.
  Вычисляемые поля (счётчики, isLiked/isFavorite/isFollowing, имена/аватары из навигаций) помечены
  `Ignore` — заполняются в сервисах фич.
- **FluentValidation.** Валидаторы для контрактных DTO: Register (совпадение паролей, email, длины),
  Login, AddLocation, UpdateLocation, AddPostComment, AddPostFavorite, UpdateUserProfile;
  регистрация `AddValidatorsFromAssemblyContaining<RegisterDtoValidator>()`.
- **DI.** `Infrastructure/DependencyInjection.cs` расширен: `Configure<JwtOptions>`,
  `AddHttpContextAccessor`, `ITokenService`/`ICurrentUserService`/`IFileService`,
  `AddAutoMapper(assembly)`, `AddValidatorsFromAssemblyContaining`.
- **WebApi.** `WebApi/Extensions/WebApiServiceExtensions.cs`: `AddJwtAuthentication` (валидация
  issuer/audience/lifetime/ключа, `ClockSkew=0`, `NameClaimType=userName`, `RoleClaimType=Role`) +
  **fallback-политика** «требуется аутентификация» (все эндпоинты защищены по умолчанию);
  `AddSwaggerWithBearer` (security scheme Bearer + глобальное требование + XML-комментарии).
- **Middleware.** `WebApi/Middleware/ExceptionHandlingMiddleware.cs` — маппинг
  ValidationException/BadRequest→400, NotFound→404, Forbidden→403, UnauthorizedAccess→401,
  прочее→500; сериализация `Response<string>` (camelCase); 5xx→LogError, 4xx→LogWarning.
- **Program.cs.** Подключены `AddInfrastructure` → `AddJwtAuthentication` → `AddSwaggerWithBearer`;
  в конвейере: exception-middleware первым, `UseAuthentication` перед `UseAuthorization`.

## Решения
- **`System.IdentityModel.Tokens.Jwt` 7.1.2 добавлен в Infrastructure.** Shared framework
  `Microsoft.AspNetCore.App` НЕ экспортирует IdentityModel-сборки в обычный проект (только
  через пакет JwtBearer) — сборка Infrastructure падала на `JwtSecurityToken`. Версия 7.1.2 точно
  совпадает с тем, что транзитивно тянет ASP.NET Core 8.0.11 JwtBearer в WebApi → без конфликта.
- **Схема JWT-аутентификации регистрируется в WebApi, а не в Infrastructure.** Пакет
  `JwtBearer` и типы `TokenValidationParameters` живут в WebApi; Infrastructure отвечает только
  за генерацию токена (`TokenService`). Чётко разделены генерация (Infra) и валидация (WebApi).
- **Авторизация по умолчанию через `FallbackPolicy`.** Реализует правило ТЗ «все эндпоинты
  защищены, кроме register/login» без `[Authorize]` на каждом контроллере — открытые эндпоинты
  помечаются `[AllowAnonymous]`. Проверено: `GET /` без токена → 401.
- **AutoMapper без `AssertConfigurationIsValid`.** Вычисляемые поля помечены `Ignore`, конфигурация
  не бросает на неотображённых членах; сложную логику (счётчики/флаги) считаем в сервисах фич.
- **FileService с параметрами allowedExtensions/maxSize.** По умолчанию — изображения/10 МБ;
  для файлов сообщений (Phase 8) достаточно передать свой набор расширений, переписывать не нужно.
