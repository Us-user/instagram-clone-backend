## Цель сессии
Реализовать Phase 18 целиком (§1): онлайн-статусы (presence) со **взаимной** приватностью
`ShowOnlineStatus` (`isOnline` + `lastSeen`) и события набора (typing) — в личном чате с именем
и `kind ∈ {text, voice}`, в группах — со списком печатающих. Всё real-time через SignalR,
эфемерное состояние в БД не пишем (кроме `LastSeen`). Держать сборку зелёной, не ломать контракт базы.

## Сделано
- **Domain — сущность:** `User.LastSeen (DateTime?)` — момент последнего онлайна (обновляется при
  уходе в офлайн). Единственное изменение схемы фазы.
- **Domain — DTO (`DTOs/Presence`):** `UserPresenceDto {UserId, IsOnline, LastSeen}`,
  `PresenceQueryDto {UserIds}` (батч-запрос), `TypingDto {ChatId, UserId, UserName, Kind}` (личное
  событие), `TypingUserDto {UserId, UserName, Kind}` + `GroupTypingDto {GroupChatId, Typers[]}`
  (групповой список). `Kind` — строка `text`/`voice` (нормализуется сервером; на проводе точно
  соответствует контракту без завязки на JsonStringEnumConverter).
- **Infrastructure — трекеры (in-memory singletons):**
  - `PresenceTracker`/`IPresenceTracker` — на пользователя набор активных `connectionId`; онлайн =
    набор непуст. `Connect` возвращает true при переходе офлайн→онлайн (первое соединение),
    `Disconnect` — true при уходе в офлайн (последнее). Синхронизация простым `lock`.
  - `TypingTracker`/`ITypingTracker` — карта `группа → (пользователь → имя/kind/expiresAt)`, TTL 6с;
    `Update` продлевает запись, лениво чистит протухшие и возвращает актуальный список печатающих.
- **Infrastructure — сервисы:**
  - `PresenceService`/`IPresenceService` — `GetStatusAsync`/`GetStatusesAsync` со **взаимной**
    логикой: если у запрашивающего `ShowOnlineStatus=false` — не видит чужие (все скрыты); статус
    цели скрыт при блокировке в любую сторону или её собственном `ShowOnlineStatus=false`; свой
    статус виден себе всегда; онлайн — из `IPresenceTracker`, `lastSeen` — из `User.LastSeen`
    (при онлайне `lastSeen=null`). `OnUserOnlineAsync`/`OnUserOfflineAsync` — переходы: офлайн
    пишет `LastSeen` (ExecuteUpdate) и рассылает; онлайн рассылает. Рассылка адресная — собеседники
    по личным чатам ∪ участники общих групп, минус блок и минус выключившие показ (взаимность).
  - `TypingService`/`ITypingService` — best-effort (без БД-записи): личный чат — проверка участия
    + блокировки, доставка собеседнику; группа — проверка членства, обновление `TypingTracker`,
    рассылка списка остальным. При недопустимом вводе молча выходит (это сигнал, не команда).
    `NormalizeKind` → `voice`/`text`.
  - Абстракции рассылки: `IPresenceNotifier`, `ITypingNotifier` (реализации — в WebApi).
- **Infrastructure — DI:** `IPresenceService`/`ITypingService` — scoped; `IPresenceTracker`/
  `ITypingTracker` — singleton (эфемерное состояние между всеми соединениями).
- **WebApi — хабы:** `PresenceAwareHub<TClient>` — базовый: `OnConnected/OnDisconnected` →
  `IPresenceTracker` + `IPresenceService` на переходах. `ChatHub`, `GroupChatHub`, `NotificationHub`
  переведены на него; добавлен новый `PresenceHub` (`/presenceHub`) + `IPresenceClient.ReceivePresence`.
  В `ChatHub`/`GroupChatHub` — клиент→сервер метод `Typing(id, kind)` (читает userId/userName из claims,
  делегирует в `ITypingService`). В `IChatClient` добавлен `UserTyping(TypingDto)`, в `IGroupChatClient`
  — `GroupTyping(GroupTypingDto)`.
- **WebApi — нотификаторы:** `PresenceNotifier` (поверх `PresenceHub`), `TypingNotifier` (поверх
  `ChatHub`/`GroupChatHub`: `UserTyping` собеседнику, `GroupTyping` — участникам). Оба scoped.
- **WebApi — контроллер:** `PresenceController` — `GET /Presence/get-status?userId`,
  `POST /Presence/get-statuses` (body `{ userIds }`).
- **WebApi — Program.cs:** регистрация `IPresenceNotifier`/`ITypingNotifier`, `app.MapHub<PresenceHub>("/presenceHub")`.
- **Сид:** `bob.LastSeen = -15 мин`, `carol.LastSeen = -26 ч` — чтобы `get-status` сразу отдавал осмысленный `lastSeen`.

## Решения
- **Онлайн — по живым соединениям, а не флагу в БД.** `IsOnline` определяется трекером активных
  SignalR-подключений; в БД хранится только `LastSeen`. Эфемерное состояние (presence, typing) —
  in-memory singletons, переживать рестарт не нужно.
- **Presence кормится всеми хабами.** Через общий `PresenceAwareHub<T>` любое соединение (чат/группа/
  уведомления/presence) делает пользователя онлайн — «онлайн» = приложение открыто (обычно висит
  notificationHub). Так статус не гаснет, когда закрыт конкретно чат.
- **Взаимность `ShowOnlineStatus` — на чтении и на рассылке.** В `GetStatuses`: скрывший себя не
  видит чужие. В рассылке: скрывший себя не транслирует статус; получатели, скрывшие себя,
  исключаются из адресатов. Плюс блокировка в любую сторону скрывает статус в обе стороны.
- **Групповой typing — весь список, а не отдельные события.** §1 требует «уходит список печатающих».
  Сервер собирает список в `TypingTracker` (TTL 6с, чуть больше клиентского авто-сброса ~3с), лениво
  чистит протухших при каждом событии и рассылает актуальный набор. Личный typing — простая
  ретрансляция одного события собеседнику.
- **`TypingService` не бросает.** Событие набора — «сигнал»: при не-участнике/блоке/чужом чате просто
  ничего не отправляем (иначе хаб вернул бы клиенту ошибку на безобидное действие).
- **Рассылка presence адресная.** Только собеседники ∪ co-members — приватность и масштаб (не broadcast всем).
- **`Kind` — строка.** Чтобы на проводе гарантированно были `text`/`voice` из контракта без завязки
  на глобальный конвертер enum→string.
