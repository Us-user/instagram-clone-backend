## Цель сессии
Реализовать Phase 16 целиком: реакции, reply, forward и голосовые сообщения — одинаково для
личных (Direct) и групповых (Group) чатов. Новый кросс-контекстный контроллер `/Message`
(react/forward/send-voice), полиморфная сущность `MessageReaction`, расширение личного `Message`
полями §8, real-time пуш реакций, миграция + сид. Держать сборку зелёной и не ломать контракт базы.

## Решения
- **Полиморфная `MessageReaction` без FK на сообщение.** Один `MessageId` не может ссылаться FK-ключом
  сразу на две таблицы (`Messages`/`GroupMessages`), поэтому целостность держит unique-индекс
  `(MessageId, MessageContext, UserId)`, а очистка при удалении сообщения/чата делается вручную
  (`ExecuteDeleteAsync`). Альтернатива с двумя nullable-FK усложнила бы модель без выгоды.
- **Реакции в DTO — через enrichment, не через проекцию.** Полиморфизм без навигации не транслируется
  в EF-выражение, поэтому реакции догружаются батч-запросом и раскладываются по сообщениям
  (`ReactionEnrichment`), как `MentionEnrichment` в Phase 13.
- **Единый `/Message` для обоих контекстов.** react/forward/voice — кросс-контекстные, поэтому вынесены
  в отдельный `MessageService`/`MessageController` с параметром `context`, а не размазаны по
  `ChatService`/`GroupChatService`. Reply для личных встроен в существующий `/Chat/send-message`
  (расширение необязательным полем), как reply в группах.
- **Голосовые в `wwwroot/voice`.** FileService обобщён параметром папки (обратная совместимость через
  необязательные аргументы); удаление voice-файлов учитывает папку по `MessageType`. Раздача — штатным
  `UseStaticFiles` (сервит все подпапки wwwroot).
- **Forward = физическая копия файла.** Пересланное сообщение независимо: файл копируется под новым
  именем (`FileService.CopyFile`), ссылку на оригинал не храним (по §8), `ReplyToMessageId` не переносим.
- **Waveform — детерминированный плейсхолдер.** Реальный анализ аудио (NAudio/FFmpeg) вне объёма фазы;
  поле всегда заполнено, генерацию можно подключить позже без изменения контракта.
- **Блокировки/приватность.** Для Direct-реакции/пересылки/голосового — блок-проверка между участниками
  (как в чате); для Group — доступ = членство. На служебные (System) сообщения реагировать и пересылать
  их нельзя.
