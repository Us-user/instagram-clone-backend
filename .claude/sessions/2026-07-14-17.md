## Решения
- **Упоминания как батч-обогащение, а не SQL-проекция.** `Mention` — полиморфная ссылка
  (`EntityType`+`EntityId`), у `Post`/`PostComment` нет прямой FK-навигации на неё, поэтому
  включить упомянутых в `PostProjections.ToDto` (Expression→SQL) нельзя. Заполняем после
  материализации одним батч-запросом (`MentionEnrichment`) по Id объектов страницы — единый
  путь для всех лент, без N+1.
- **`WhoCanMention` = Followers → «упоминающий подписан на адресата».** Аудитория «Подписчики»
  = те, кто на меня подписан, поэтому автор упоминания должен быть одобренным подписчиком
  адресата (`AccessGuard.IsAcceptedFollowerAsync(author, target)`). Nobody — запрещено всем,
  Everyone — всем. Настройка читается из `PrivacySettings` (нет строки → default Everyone).
- **При запрете упоминания запись не создаётся** (и уведомление тоже) — выбран строгий вариант
  §4 («упоминание не создаётся»), поэтому в `MentionedUsers` попадают только реально созданные
  упоминания (блок/приват-фильтр не утекает существование адресата).
- **`PostsCount` денормализован** и правится сервисом (инкремент при добавлении тега к посту,
  декремент при удалении). Декремент — до `Posts.Remove` (иначе каскад уже снёс бы связи).
- **Trending за окно 7 дней** (число постов с тегом за период, tie-break по общему `PostsCount`)
  — отделяет «тренды за период» от «поиска по популярности» (§3).
- **Edit поста:** в базовом контракте нет эндпоинта редактирования поста, поэтому парсинг тегов/
  упоминаний навешен только на создание; при появлении edit — переиспользовать те же методы.
