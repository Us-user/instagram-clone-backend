Ты завершаешь текущую рабочую сессию над бэкендом Instagram-клона.
Требования: база — `instagram-backend-prompt.md` (готово), новые фичи — `instagram-backend-features-prompt.md` (текущая работа, Phase 11+).

## 1. Оценить состояние
- Выполни `git status` и `git diff --stat` — что изменилось за сессию.
- Прочитай последний лог в `.claude/sessions/` (если есть), чтобы не дублировать.
- При необходимости запусти `dotnet build`, чтобы зафиксировать реальное состояние сборки в логе.

## 2. Сохранить лог сессии
- Определи сегодняшнюю дату и создай/дополни лог `.claude/sessions/<YYYY-MM-DD>-NN.md` (NN = 01, 02… — увеличивай, если файл на сегодня уже есть) по шаблону из `.claude/sessions/README.md`:
  - что сделано за сессию, принятые решения, затронутые файлы, состояние сборки, открытые TODO, следующие шаги для `/start`.
- В поле `phase` укажи активную фазу роадмапа (напр. `Phase 11 — Уведомления`).
- Обнови в `ROADMAP.md` блок **📍 Текущий статус** и чекбоксы задач под реальное положение дел.

## 3. Закоммитить
- `git add -A`
- `git commit -m "<фаза>: <краткое описание>"`
- Если коммитить нечего — явно скажи об этом.

## 4. Отчитаться
- Сообщи пользователю: что закоммичено (хеш + сообщение коммита), путь к логу сессии и где сейчас стоит проект.
