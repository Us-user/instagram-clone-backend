Ты возобновляешь работу над бэкендом Instagram-клона. Источник истины по требованиям — `instagram-backend-prompt.md`.

## 1. Загрузить контекст
1. Прочитай `ROADMAP.md` целиком — найди активную фазу и ближайшие невыполненные задачи `[ ]`.
2. Прочитай `CLAUDE.md` — конвенции проекта.
3. Найди логи в `.claude/sessions/` (`*.md`, кроме `README.md`), отсортируй по имени по убыванию и прочитай последние 1–3 — что делали в прошлый раз, какие открытые TODO / решения.
4. Выполни `git log --oneline -10` и `git status`, чтобы увидеть состояние репозитория.
5. Если пользователь передал аргумент `$ARGUMENTS` — учти его как фокус этой сессии.

## 2. Спланировать сессию
- Коротко сообщи: текущая фаза, что было сделано в прошлый раз, и конкретные задачи на эту сессию (бери следующий логичный связный кусок — обычно завершение текущей фазы или ясную под-цель).
- Если в репозитории остались незакоммиченные изменения после прошлого раза — сначала разберись с ними.

## 3. Реализовать
- Делай работу строго по слоистой архитектуре и ТЗ (пути, методы, DTO — дословно).
- Держи сборку зелёной: после значимых изменений запускай `dotnet build` и чини ошибки.
- Обновляй `ROADMAP.md`: отмечай `[x]` выполненное, `[~]` в работе, и обнови блок **📍 Текущий статус** (активная фаза, дата последней сессии, следующий шаг, состояние сборки).

## 4. Закрыть сессию (ВСЕГДА перед завершением)
1. Определи сегодняшнюю дату и создай новый лог `.claude/sessions/<YYYY-MM-DD>-NN.md` (NN = 01, 02… — увеличивай, если файл на сегодня уже есть) по шаблону из `.claude/sessions/README.md`: что сделано, решения, затронутые файлы, состояние сборки, открытые TODO, следующие шаги.
2. Зафиксируй изменения:
   - `git add -A`
   - `git commit -m "<фаза>: <краткое описание сессии>"`
3. Отчитайся пользователю: что завершено, прогресс по роадмапу, что подхватит `/start` в следующий раз.
